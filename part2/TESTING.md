# HBnB Part 2 Testing Report

## Objective

This report documents the validation and testing completed for the HBnB
Part 2 REST API. The goal is to confirm that the user, amenity, place,
and review endpoints follow the expected input formats, output formats,
status codes, and validation rules.

## Validation Covered

- User first name, last name, and email are required.
- User email must use a valid format and must be unique.
- Amenity name is required.
- Place title is required.
- Place price must be a number greater than or equal to zero.
- Place latitude must be between -90 and 90.
- Place longitude must be between -180 and 180.
- Place owner must reference an existing user.
- Place amenities must reference existing amenities.
- Review text is required.
- Review rating must be an integer between 1 and 5.
- Review user and place must reference existing entities.

## Swagger Documentation

Swagger documentation is generated by Flask-RESTx and is available when
the application is running:

```text
http://127.0.0.1:5000/api/v1/
```

## Manual cURL Test Plan

Start the application:

```bash
python3 run.py
```

### Create a User

```bash
curl -X POST http://127.0.0.1:5000/api/v1/users/ \
  -H "Content-Type: application/json" \
  -d '{"first_name": "Jane", "last_name": "Doe", "email": "jane.doe@example.com"}'
```

Expected status: `201 Created`

### Reject Invalid User Data

```bash
curl -X POST http://127.0.0.1:5000/api/v1/users/ \
  -H "Content-Type: application/json" \
  -d '{"first_name": "", "last_name": "", "email": "invalid-email"}'
```

Expected status: `400 Bad Request`

### Create an Amenity

```bash
curl -X POST http://127.0.0.1:5000/api/v1/amenities/ \
  -H "Content-Type: application/json" \
  -d '{"name": "Wi-Fi"}'
```

Expected status: `201 Created`

### Create a Place

Replace `<user_id>` and `<amenity_id>` with IDs returned by earlier
requests.

```bash
curl -X POST http://127.0.0.1:5000/api/v1/places/ \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Cozy Apartment",
    "description": "A nice place to stay",
    "price": 100.0,
    "latitude": 37.7749,
    "longitude": -122.4194,
    "owner_id": "<user_id>",
    "amenities": ["<amenity_id>"]
  }'
```

Expected status: `201 Created`

### Reject Invalid Place Coordinates

```bash
curl -X POST http://127.0.0.1:5000/api/v1/places/ \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Bad Place",
    "description": "Invalid coordinates",
    "price": 50.0,
    "latitude": 100,
    "longitude": 200,
    "owner_id": "<user_id>",
    "amenities": []
  }'
```

Expected status: `400 Bad Request`

### Create a Review

Replace `<user_id>` and `<place_id>` with existing IDs.

```bash
curl -X POST http://127.0.0.1:5000/api/v1/reviews/ \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Great stay!",
    "rating": 5,
    "user_id": "<user_id>",
    "place_id": "<place_id>"
  }'
```

Expected status: `201 Created`

### Reject Invalid Review Rating

```bash
curl -X POST http://127.0.0.1:5000/api/v1/reviews/ \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Bad rating",
    "rating": 6,
    "user_id": "<user_id>",
    "place_id": "<place_id>"
  }'
```

Expected status: `400 Bad Request`

### Get Reviews for a Place

```bash
curl http://127.0.0.1:5000/api/v1/places/<place_id>/reviews
```

Expected status: `200 OK`

### Delete a Review

```bash
curl -X DELETE http://127.0.0.1:5000/api/v1/reviews/<review_id>
```

Expected status: `200 OK`
