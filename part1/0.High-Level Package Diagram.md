# Database Entity Relationship Diagram (ERD)


This Entity Relationship Diagram (ERD) represents the database structure of the HBnB system. It illustrates the system's entities, their attributes, and the relationships between them. The diagram provides a clear view of how data is organized, stored, and connected to support the application's functionality.

## Main Entities

The database is built around the core entities of the HBnB system, including Users, Owners, Places, Bookings, Reviews, Amenities, and Notifications. It also contains supporting entities such as Countries, Cities, Place Types, Room Details, Availability, Seasonal Pricing, Cancellation Policies, Activity Categories, Booking History, Review Analytics, Guest Reviews, and Review Reports. A junction table (Place_Amenity) is used to manage the many-to-many relationship between Places and Amenities.

## Relationships

The entities are connected through one-to-many and many-to-many relationships using primary keys (PK) and foreign keys (FK). For example, an Owner can manage multiple Places, a User can create multiple Bookings and Reviews, and each Place can have multiple Amenities, Reviews, and Booking records. These relationships ensure data consistency while accurately representing the business rules of the HBnB system.

## Design Decisions

The database follows a normalized relational design to minimize data redundancy and improve maintainability. Each table stores a specific type of information, while primary keys uniquely identify records and foreign keys enforce referential integrity between related tables. This design improves scalability and makes the database easier to maintain and extend as the application evolves.
