# Database Entity Relationship Diagram (ERD)

## Purpose

This Entity Relationship Diagram (ERD) shows the database structure of the HBnB system. It illustrates the main entities, their attributes, and the relationships between them. The diagram helps explain how the application's data is organized and connected.

## Main Entities

The database includes the main entities used in the system, such as Users, Owners, Places, Bookings, Reviews, Amenities, and Notifications. It also contains supporting entities for locations, room details, availability, seasonal pricing, and cancellation policies.

## Relationships

The relationships define how the entities interact with each other. For example, an Owner can manage multiple Places, a User can create Bookings and Reviews, and each Place can have multiple Amenities, Reviews, and Bookings. These relationships help maintain data consistency and support the application's business logic.

## Design Decisions

The database is designed using primary keys (PK) and foreign keys (FK) to maintain referential integrity. Related information is separated into different tables to reduce data duplication and improve scalability, making the database easier to maintain and extend.
