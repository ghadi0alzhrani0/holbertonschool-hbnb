### Flow 1: User Authentication (Login)
```mermaid
sequenceDiagram
    autonumber
    title Flow 1: User Authentication (Login)

    actor User as User / Owner
    participant FE as Frontend (UI / JS)
    participant BE as Backend API (Server)
    participant DB as Database

    User->>FE: Fill Login Form & Click "Submit"
    FE->>BE: POST /api/auth/login (email, password, role)
    
    alt If Role is Owner
        BE->>DB: SELECT * FROM owners WHERE email = input_email
        DB-->>BE: Return Owner Record (hashed_password)
    else If Role is Guest
        BE->>DB: SELECT * FROM users WHERE email = input_email
        DB-->>BE: Return Guest Record (hashed_password)
    end

    BE->>BE: Verify Password (inputPassword, storedPassword)
    
    alt Credentials Valid
        BE->>BE: Generate JWT Token (actorId, roleType)
        BE-->>FE: HTTP 200 OK (Token & Redirect URL)
        FE->>User: Display Dashboard & Store Token
    else Credentials Invalid
        BE-->>FE: HTTP 401 Unauthorized (Error Message)
        FE->>User: Display Invalid Credentials Alert
    end

    sequenceDiagram
    autonumber
    title Flow 3: End-to-End Booking Lifecycle

    actor Guest as Guest User
    participant FE as Frontend App
    participant BE as Backend API (Server)
    participant DB as Database
    actor Owner as Property Owner

    %% Phase 1: Guest Request
    Guest->>FE: Select Dates & Click "Book Now"
    FE->>BE: POST /api/bookings (placeId, startDate, endDate)
    BE->>BE: Validate Date Constraints
    
    BE->>DB: SELECT * FROM bookings WHERE placeId = id AND overlapping dates
    DB-->>BE: Availability Status (isBooked = False)
    
    BE->>DB: SELECT special_price FROM pricing WHERE placeId = id
    DB-->>BE: Return Seasonal Price Details
    
    BE->>BE: Calculate Total Cost (basePrice, specialPrice, duration)
    
    BE->>DB: INSERT INTO bookings (guestId, status) VALUES (id, 'Pending')
    DB-->>BE: Return bookingId & status='Pending'
    
    BE-->>FE: HTTP 201 Created (Booking Pending Approval)
    FE->>Guest: Display: Booking Placed! Waiting for Owner Approval

    %% Phase 2: Owner Action
    Owner->>FE: Open Dashboard & Click "Approve Booking"
    FE->>BE: PUT /api/bookings/{id}/status (status='Confirmed')
    BE->>BE: Process Booking Decision Rules
    
    BE->>DB: UPDATE bookings SET status = 'Confirmed' WHERE id = bookingId
    DB-->>BE: Status Updated Successfully
    
    BE->>DB: INSERT INTO booking_history (bookingId, transition) VALUES (id, 'Pending->Confirmed')
    DB-->>BE: History Logged
    
    BE->>DB: INSERT INTO notifications (userId, message) VALUES (guestId, 'Your booking is confirmed!')
    DB-->>BE: Notification Staged
    
    BE-->>FE: HTTP 200 OK (Booking Confirmed Successfully)
    FE->>Owner: Refresh View & Display Status Updated
