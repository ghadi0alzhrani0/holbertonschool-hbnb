# Business Logic Class Diagram

This Class Diagram represents the Business Logic layer of the HBnB system. It illustrates the main classes, their attributes, methods, and relationships, showing how the application's business objects work together to support the system's functionality.

## Main Classes

The Business Logic layer consists of core classes such as **User**, **Owner**, **Place**, **Booking**, **Review**, **Amenity**, and **SystemNotification**, which handle the main features of the system. Supporting classes including **Country**, **State**, **City**, **PlaceType**, **CancellationPolicy**, **RoomDetail**, **PlaceAvailability**, **SeasonalPricing**, **AmenityCategory**, **BookingGuest**, **BookingHistory**, **ReviewRatingDetails**, **ReviewResponse**, and **GuestReview** manage location data, property details, booking information, pricing rules, review management, and notifications.

## Relationships

The classes are connected through associations and compositions to represent the business rules of the HBnB system. For example, an **Owner** can manage multiple **Places**, a **User** can create multiple **Bookings** and **Reviews**, and a **Place** contains room details, availability schedules, seasonal pricing, amenities, bookings, and reviews. Additional relationships connect bookings with guest information and booking history, reviews with rating details and owner responses, and users or owners with system notifications.

## Design Decisions

The Business Logic layer follows an object-oriented design where each class has a single responsibility. Attributes store the required data, methods define the main business operations, and relationships organize how objects interact with one another. Supporting classes are separated from the core classes to improve readability, reduce complexity, and make the system easier to maintain and extend.
